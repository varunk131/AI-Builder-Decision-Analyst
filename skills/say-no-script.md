# Say No Script — Stakeholder Pushback Generator

This skill helps you construct a clear, defensible, relationship-preserving no. It first validates whether saying no is the right call, then gives you 3 scripts for different approaches.

## PHASE 1: Input Collection

The PM describes a stakeholder request they need to push back on.

## PHASE 2: Validation Check

Before scripting the no, validate:

| Check | Question |
|-------|----------|
| **Signal Check** | Is the requestor seeing something you're not? |
| **Political Check** | Is this battle worth fighting politically? |
| **Compromise Check** | Is there a version of yes that works? |

**Verdict:** [Say no / Say yes differently / Accept this one]

If verdict is "Accept this one," explain why and stop. Otherwise proceed to Phase 3.

## PHASE 3: Three Scripts

### Script 1: The Redirect
Says no to the REQUEST while saying yes to the NEED.

| Beat | Content |
|------|---------|
| Alignment | Show you understand the underlying need |
| Reframe | Redirect to the real problem |
| Alternative | Offer a different solution |
| Commitment | Specific next step with a date |

### Script 2: The Data Shield
Uses data to remove emotion from the conversation.

| Beat | Content |
|------|---------|
| Shared data | Present the numbers both sides can see |
| Explicit trade-off | Name what saying yes costs |
| Make them choose | Force the prioritization decision |

### Script 3: The Time Buy
Creates space to avoid snap decisions under pressure.

| Beat | Content |
|------|---------|
| Acknowledge urgency | Validate their timeline pressure |
| Introduce question | Surface one thing that needs answering first |
| Propose investigation | Timeboxed research before committing |
